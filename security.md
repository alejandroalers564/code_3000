# Security Policy

## Intended Users
The intended users of this repository are the student (me) and the course instructional staff for grading purposes.

## Risk Assessment
The primary security risk is academic integrity (plagiarism) rather than data theft. There is also a risk of accidental data loss or "force pushing" which could delete assignment history.

## Security Measures
To secure this repository, I have implemented the following:
* **GitHub Ruleset:** An active ruleset named "Assignments-Protection" is applied to the main branch.
* **Protection Measures:** I have enabled "Block force pushes" and "Require a pull request before merging" to prevent accidental direct updates.
* **CODEOWNERS:** A CODEOWNERS file is used to define ownership of the codebase.
